## 一、JS 基础原理 ⭐⭐⭐

### 1. call / apply / bind

**核心思路：** 把函数挂到目标对象上借用 this，执行完再删掉。

```
context = context ?? globalThis → Object(context)
用 Symbol 挂 fn → 执行 → delete
bind 额外处理：new 调用时 this 指向实例而非绑定的 context
bind 还要继承原型链：boundFn.prototype = Object.create(targetFn.prototype)
```

**追问：**
- apply 和 call 区别？参数传法不同，apply 传数组
- bind 返回的函数被 new 时，绑定的 context 会被忽略，this 指向新实例

---

### 2. new

**核心思路：** 4 步走。

```
1. Object.create(Constructor.prototype)  // 创建 + 链接原型
2. Constructor.apply(obj, args)          // 绑定 this 执行
3. 判断返回值：是对象/函数 → 返回它，否则 → 返回 obj
```

**追问：**
- 为什么要 Object.create？让实例能访问原型方法
- 箭头函数为什么不能 new？没有自己的 this，没有 prototype

---

### 3. instanceof

**核心思路：** 沿原型链向上找，看能不能碰到 `right.prototype`。

```
排除左侧基本类型 → Object.getPrototypeOf 循环向上 → 找到返回 true，到 null 返回 false
```

**追问：**
- `[] instanceof Object` 为什么是 true？原型链：`[] → Array.prototype → Object.prototype`
- `1 instanceof Number` 为什么是 false？1 是基本类型，不是对象实例

---

### 4. 防抖 / 节流

**防抖核心：** 每次触发都重置定时器，只执行最后一次。

```
clearTimeout(timer) → timer = setTimeout(fn, delay)
```

**节流完美版（首次立即 + 末尾补全）：**

```
remain = delay - (now - lastTime)
remain <= 0 → 立即执行，更新 lastTime，清掉已有定时器
remain > 0 && !timer → 设 remain 后的定时器补最后一次
```

**追问：**
- 防抖场景：搜索输入、表单校验、resize
- 节流场景：scroll、mousemove、拖拽
- 为什么要 apply？保留调用时的 this 和参数

---

### 5. 深拷贝

**核心思路：** 递归 + WeakMap 处理循环引用 + 特殊类型单独处理。

```
基本类型直接返回
map.has(obj) → 返回缓存（处理循环引用）
Date → new Date(obj)
RegExp → new RegExp(obj.source, obj.flags)
Array.isArray → [] 否则 {}
for...in + hasOwnProperty 递归每个属性
```

**追问：**
- 为什么不用 JSON.parse(JSON.stringify())？循环引用报错，Date/RegExp/函数丢失
- WeakMap vs Map？WeakMap 键只能是对象，弱引用，不阻止 GC 回收
- for...in 有什么问题？会遍历原型链属性，要配合 hasOwnProperty

## 二、Promise 系列 ⭐⭐⭐

### 6. Promise.all

```
返回 Promise → 遍历用 Promise.resolve 包装
result[index] = value，count++ → count === length 时 resolve
任一 catch → 直接 reject
空数组 → resolve([])
```

---

### 7. Promise.allSettled

```
和 all 区别：不会 reject
成功存 {status:'fulfilled', value}
失败存 {status:'rejected', reason}
用 finally 统一计数，全部落定后 resolve(result)
```

---

### 8. Promise.race

```
谁先落定谁决定结果
for...of 遍历，Promise.resolve 包装
空数组永远 pending（循环内什么都没做）
```

---

### 9. Promise.any

```
一个成功就 resolve
全失败才 reject(new AggregateError(errors, '...'))
空数组直接 reject（与 all 空数组 resolve 相反）
```

| 方法 | 成功条件 | 失败条件 | 空数组 |
|---|---|---|---|
| all | 全部成功 | 一个失败 | resolve([]) |
| race | 谁先 settled | 谁先失败 | pending |
| any | 一个成功 | 全失败 | reject |
| allSettled | 全部完成 | 不会失败 | resolve([]) |

---

### 10. 限并发调度器

**核心思路：** 先启动 min(limit, length) 个任务，谁完成谁补位。

```
taskIndex 记录下一个要执行的任务
curIndex 在发起前保存（异步闭包陷阱，必须在异步前拿到）
完成后 count++ → count === length 时 resolve，否则 runTask() 补位
Promise.resolve().then(() => curTask()) 兜住同步报错
```

## 三、数据结构与算法 ⭐⭐

### 11. 数组转树 / 树转数组

**数组转树（O(n)）：**

```
第一遍：Map 存所有节点（带 children:[]）
第二遍：parentId === 0 → 根节点 push 到 tree
        否则 → map.get(parentId).children.push(node)
```

**树转数组（DFS）：**

```
解构 {children=[], ...rest}，push rest，递归 children
不要 delete item.children（会改原树）
```

---

### 12. LRU 缓存

**核心思路：** Map 天然维护插入顺序，命中时先删再重新插入刷新顺序。

```
get：has → 取值 → delete → set → return value；否则 return -1
put：已存在 → delete → set；超容量 → 删 map.keys().next().value → set；否则直接 set
```

**追问：**
- 为什么用 Map 不用对象？Map 保证插入顺序，且 keys().next() 能拿到最旧的 key

---

### 13. 归并排序

```
递归二分 → 单元素天然有序 → 双指针合并两个有序数组
时间复杂度 O(n log n)，稳定排序
```

## 四、设计模式 ⭐⭐

### 14. 发布订阅（EventEmitter）

```
Map 存 eventName → callbacks[]
on:   push callback
emit: 拷贝数组后遍历（防止 emit 中 off 导致遍历错乱）
off:  filter 掉指定 callback
once: 包一层 wrapper，触发后先 off(wrapper) 再执行原 callback
```

**追问：**
- emit 为什么要拷贝数组？防止回调中调用 off 修改数组导致遍历跳项
- once 为什么包一层？off 需要引用同一个函数才能移除，直接传 callback 无法移除

---

### 15. 单例模式

```
static instance = null
static getInstance() → !instance 时才 new，否则返回已有实例
```

---

### 16. 观察者 vs 发布订阅

```
观察者：Subject 直接持有 Observer 列表，状态变化直接通知，耦合较紧
发布订阅：通过事件中心解耦，发布者和订阅者互不认识（EventEmitter 就是发布订阅）
```

## 五、工程实用题 ⭐⭐

### 17. 洋葱模型（Koa compose）

**核心思路：** 递归 dispatch，用 index 防止同一中间件多次调用 next。

```
dispatch(i) → 取 middlewares[i]，传入 ctx 和 () => dispatch(i+1)
i <= index → reject（防重复调用 next）
Promise.resolve 包裹兼容普通函数和 async 函数
```

---

### 18. 防竞态

```
维护 currentId，每次请求前 id = ++currentId
请求完成后 id !== currentId → 丢弃结果，return undefined
```

---

### 19. 请求缓存

```
Map 存 key → promise
请求发出时立即缓存（不是成功后），复用 pending 中的 promise
失败时 cache.delete(key) 清除缓存，下次重新请求
key 用 JSON.stringify(args) 生成
```

---

### 20. retry 重试

**Promise 版（递归）：**

```
.catch 里判断 times <= 1 则 throw，否则 await delay → 递归 retry(fn, times-1)
```

**async 版（循环）：**

```
for 循环 try/catch，最后一次（i === times）才 throw，否则 await delay
```

## 六、数组方法 ⭐⭐

### 21. 数组扁平化

```
遍历：Array.isArray(item) && depth > 0 → 递归展开，否则直接 push
reduce 版：acc.concat(Array.isArray(cur) ? flatten(cur, depth-1) : cur)
```

---

### 22. 数组去重

```
基本类型：Array.from(new Set(arr))
引用类型：Map 存 item[key]，filter 时判断 !map.has 且 map.set
indexOf 版：arr.filter((item, i, self) => self.indexOf(item) === i)
```

---

### 23. map / filter / reduce

**map：** `new Array(len)` 保留稀疏空洞，`i in arr` 判断跳过空位，按索引赋值

**filter：** push 不保留空洞，返回新数组

**reduce：** 无初始值时找第一个有效元素；空数组无初始值报错；用 `arguments.length >= 2` 判断是否传了初始值

## 七、加分项 ⭐

### 24. LazyMan

```
任务队列 + next 串行调度
关键：Promise.resolve().then(() => this.next()) 延迟启动，等链式调用收集完任务再执行
sleepFirst 用 unshift 插队到队列头部
```

---

### 25. 红绿灯

```
wait(fn, time) 封装成返回 Promise 的定时器
async/await + while(true) 循环顺序执行
```

---

### 26. 模板字符串渲染

```
/\{\{(\w+)\}\}/g 正则 + replace 回调
key in data 取值，取不到返回空字符串
嵌套路径支持：对 key 做 split('.') 逐层取值
```

---

### 27. lodash.get / set

```
path 统一转数组：replace /\[(\w+)\]/g → ".$1"，再 split(".").filter(Boolean)
get：逐层取值，遇到 null/undefined 返回 defaultValue
set：逐层取值，中间层不存在时按下一个 key 是否为数字决定创建 [] 还是 {}
```

---

### 28. useEvent（React Hook）

```
useRef 存最新 fn
useLayoutEffect 每次渲染后同步 fnRef.current = fn（DOM 更新后、绘制前）
useCallback([]) 返回引用稳定的函数，内部调 fnRef.current(...args)
```

**追问：**
- 为什么不直接用 fn？会读到旧闭包
- 为什么用 useLayoutEffect 不用 useEffect？避免绘制前后逻辑不一致

---

### 29. 手写 Ajax

```
new XMLHttpRequest → open(method, url, true) → setRequestHeader
onreadystatechange：readyState === 4 && status 2xx → success
ontimeout / onerror → error
GET 参数拼 URL，POST 根据 Content-Type 决定 send 内容
```
